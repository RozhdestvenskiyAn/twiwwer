6. [ ] Оценивать фразы других пользователей

# _Модель данных_
### User 
_id, nickname, password, email, time_insert_

# _Функиональные требования_
## _1.1. Метод registration_

**Входящие данные** <br>

```json
{
  "nickname": "ivanov2000",
  "password": "ivan2000pass"
}
```

**Валидация** <br>
nickname >= 4 символа, <= 15 символов, A-Za-z0-9._- <br>
nickname >= 8 символа, <= 100 символов, A-Za-z0-9.,:;_-?!+=/'\"*[]{}() <br>

**Логика метода** <br>
Валидация входных данных, в случае нарушений, возвращает код ошибки "REQUEST_VALIDATION_ERROR". 
Поиск пользователя в талице User по столбцу nickname, если такой nickname существует в таблице User - 
возвращает код ошибки "NICKNAME_BUSY", иначе генерируется access_token, шифрование password, сохрание пользователя
в таблицу User и возвращает accessToken зарегистрированного пользователя.

**Исходящие данные в случае успеха** <br>
статус 200
```json
{
  "data": {
    "accessToken": "fkd3243904JhdHsd231SSFDF2"
  }
}
```

**Исходящие данные в случае ошибки** <br>
статус 400
```json
{
  "error": {
    "code": "REQUEST_VALIDATION_ERROR",
    "message": "Не корректный nickname или password"
  }
}
```
```json
{
  "error": {
    "code": "NICKNAME_BUSY",
    "message": "Пользователь с таким ником уже существует"
  }
}
```
